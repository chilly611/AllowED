CREATED: scripts/test_benefits_intake.py

PURPOSE:
  End-to-end test for the VA Benefits Intake PDF generation and submission flow.
  Demonstrates the full pipeline from student data → certification PDF → 
  Benefits Intake API submission (mock or live).

FEATURES:
  ✓ Connects to Supabase and fetches Daniel Bahena's enrollment data
    - Student info: name, DOB, student ID, benefit chapter, program
    - Enrollment: term, institution, status
    - Courses: enrolled courses with units, modality, certifiable status

  ✓ Generates a professional VA enrollment certification PDF using reportlab
    - Student Information section
    - Enrollment Period details
    - Credit Hours breakdown (residential, distance, total)
    - Training Time & Rate of Pursuit
    - Course Schedule table (course ID, units, certifiable, modality)
    - Certification statement with SCO signature line
    - Professional styling with AllowED brand colors (dark #0F172A, blue #1E40AF)

  ✓ Simulates Benefits Intake API submission (2-step process)
    Step 1: POST /uploads → get pre-signed upload URL + GUID
    Step 2: PUT PDF to URL with multipart metadata + content

  ✓ Operates in MOCK MODE by default (sandbox keys are placeholders)
    - Saves PDF to output/ directory
    - Prints what would be sent to VA API
    - No actual API calls made

  ✓ Will operate in LIVE MODE when real sandbox keys are added to .env
    - Submits to https://sandbox-api.va.gov/services/vba_documents/v1/
    - Polls for processing status

DEPENDENCIES:
  - supabase (already installed via requirements.txt)
  - reportlab (already installed via requirements.txt)
  - requests (already installed via requirements.txt)
  - python-dotenv (already installed via requirements.txt)

USAGE:
  python3 scripts/test_benefits_intake.py

OUTPUT:
  Generates: output/Daniel_Bahena_Enrollment_Certification.pdf
  Example size: ~3.8 KB
